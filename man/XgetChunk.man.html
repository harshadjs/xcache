NAME

XgetChunk − Get chunk in memory.

SYNOPSIS

#include <xcache.h>

int XgetChunk(int xcacheSock, void *buf, size_t buflen, sockaddr_x *addr, socklen_t addrlen, int *flags);

DESCRIPTION

XgetChunk() fetches the chunk with address addr from the XIA network into the buffer pointed to by buf. At the max, buflen bytes are copied. If the chunk is of size greater than buflen, then the rest of the bytes are discarded and XCF_OVERFLOW is set in *flags. The argument xcacheSock must be a file descriptor returned by xcacheSocket(8). The argument addrlen must be set to the length of the address of the chunk pointed to by addr. By default, arrived chunks are not cached by the xcache. But, this behaviour can be overridden by setting XCF_DOCACHE (See below).

The argument flags is both input as well as output argument. Before the call, flags can be set by bitwise-OR’ing one or more of the following options:

XCF_DOCACHE

Overrides no-cache behaviour i.e. on arrival of a chunk the chunk is cached by local xcache.

On return, one or more of the following options can be set:

XCF_OVERFLOW

Chunk was too big to fit in the supplied buffer.

RETURN VALUE

On success, the number of bytes successfully copied is returned. On failure, -1 is returned and appropriate errno is set.

ERRORS

ENOTCONN

XcacheSock is not connected.

EHOSTDOWN

Xcache is not running.

EXAMPLE

See XrequestChunk(8).

SEE ALSO

XopenChunk(8), Xput(8), Xget(8)
