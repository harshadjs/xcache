NAME

XcacheNotifications − Xcache Notifications.

SYNOPSIS

#include <xcache.h>

int XregisterNotification(int events, void (*func)(int event, const char *cid, int cidlen));
int XprocessNotification(int xcacheSock);

DESCRIPTION

Xcache Notifications is a mechanism used by Xcache Daemon to notify applications about the happening of events. Applications register to the events of their interest by calling XregisterNotification function. The argument event is the event for which the application is registering. Following events are supported.

XCE_CHUNKARRIVED

Chunk arrived.

XCE_CHUNKEVICTED

Chunk evicted.

XCE_SLICEEXPIRED

Slice expired.

Applications can register to multiple events by a single call to XregisterNotification(). This achieved by setting events to bitwise-OR of multiple events. The argument func is a function pointer to the function that needs to be invoked on happening of any event in set events. func knows about the exact event in its first argument event. func knows about the CID of the chunk on which that particular event happened as the second argument cid. cid is of length cidlen.

Xcache sends these notifications on ALL the sockets allocated by XcacheSocket(8). Normally, applications should not need more than one such socket. Application does not need to parse and process any notification by itself. Function XprocessNotification() parses the incoming notification and invokes appropriate registered function. xcacheSock must be allocated XcacheSocket() call.

RETURN VALUE

On success XregisterNotification() returns 0. On failure it returns -1. XprocessNotification() returns the number of notifications received.

ERRORS

ENOTCONN

XcacheSock is not connected.

EHOSTDOWN

Xcache is not running.
